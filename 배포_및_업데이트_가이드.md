# 재고관리 시스템 - 배포 및 업데이트 가이드

## 전체 흐름 요약

```
코드 수정 → constants.py에서 버전 올리기 → GitHub에 올리기 → 
태그 생성 → GitHub Actions가 자동으로 EXE 빌드 → 
팀원 프로그램 실행 시 자동 업데이트 알림
```

---

## 1단계: GitHub 계정 및 저장소 준비

### 1-1. GitHub 계정 만들기
- https://github.com 접속 → Sign up

### 1-2. 새 저장소 만들기
1. GitHub 로그인 후 우측 상단 **+** 클릭 → New repository
2. Repository name: `inventory-management` (영문, 공백 없이)
3. **Public** 선택 (Private이면 자동 업데이트 시 인증 필요)
4. Create repository 클릭

### 1-3. constants.py에 GitHub 정보 입력
`core/constants.py` 파일 열어서 아래 두 줄 수정:
```python
GITHUB_OWNER = "본인의_GitHub_아이디"   # 예: "hsj1991"
GITHUB_REPO  = "inventory-management"   # 저장소 이름
```

---

## 2단계: 코드를 GitHub에 올리기 (최초 1회)

### Git이 없다면 먼저 설치
- https://git-scm.com → Download for Windows → 설치 (기본값으로)

### 터미널(명령 프롬프트)에서 실행
```
cd "C:\Users\user\Desktop\클로드 코워크"
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/본인아이디/inventory-management.git
git push -u origin main
```

---

## 3단계: 첫 번째 버전 배포하기

### 3-1. 버전 태그 붙이기
```
git tag v2.0.0
git push origin v2.0.0
```

### 3-2. GitHub Actions 자동 빌드 확인
1. GitHub 저장소 페이지 → **Actions** 탭 클릭
2. "Build and Release EXE" 워크플로우가 실행 중인지 확인
3. 약 5~10분 후 완료 → **Releases** 탭에 `재고관리시스템.exe` 파일 자동 업로드

### 3-3. EXE 파일 팀원에게 배포
1. GitHub 저장소 → **Releases** → `재고관리시스템.exe` 다운로드
2. 팀원에게 이 EXE 파일 전달
3. 팀원은 EXE를 실행하기 전에 **구글 서비스 계정 JSON 파일**과 **config.json**도 같은 폴더에 넣어야 함

### 팀원 폴더 구성
```
[팀원 PC 임의 폴더]
├── 재고관리시스템.exe          ← 배포한 파일
├── primeval-span-XXXX.json    ← 구글 서비스 계정 키 파일 (공유)
└── config.json                ← 최초 실행 시 설정 메뉴에서 자동 생성
```

---

## 4단계: 업데이트 배포하기 (이후 반복)

### 4-1. 코드 수정 후 버전 올리기
`core/constants.py` 에서 버전 번호 수정:
```python
APP_VERSION = "2.1.0"   # 2.0.0 → 2.1.0
APP_BUILD_DATE = "2026-05-01"
```

### 4-2. GitHub에 업로드 및 배포
```
git add .
git commit -m "v2.1.0: 기능 추가 내용 설명"
git push origin main
git tag v2.1.0
git push origin v2.1.0
```

### 4-3. 팀원 자동 업데이트
- 팀원이 프로그램 실행 시 3초 후 자동으로 새 버전 감지
- 팝업으로 업데이트 여부 물어봄
- "예" 클릭 시 자동 다운로드 → 교체 → 재시작

## 자주 묻는 질문

**Q. 팀원이 구글 시트 JSON 파일을 어디에 놓아야 하나요?**
> EXE 파일과 같은 폴더에 놓고, 프로그램 첫 실행 시 설정 메뉴에서 JSON 파일 경로와 구글 시트 URL을 입력하면 됩니다.

**Q. GitHub Actions 빌드가 실패하면?**
> Actions 탭에서 실패한 워크플로우를 클릭해서 오류 메시지를 확인하세요. 대부분은 requirements.txt의 패키지 오류입니다.

**Q. Private 저장소를 쓰고 싶어요**
> Private 저장소를 사용하면 자동 업데이트 API 호출 시 인증이 필요해서 복잡해집니다. 소스코드 보안이 필요하다면 별도로 문의해 주세요.

**Q. EXE 용량이 너무 커요**
> PyInstaller 특성상 Python 런타임 전체가 포함되어 보통 30~80MB 정도입니다. 정상입니다.
